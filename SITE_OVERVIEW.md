# Hi Playground - 사이트 구조 및 주요 내용 문서

> **작성일**: 2024년  
> **목적**: 현재까지의 사이트 구조, 기능, 게임 목록, 시스템 등을 종합적으로 기록

---

## 📋 프로젝트 개요

**Hi Playground**는 초등학생을 대상으로 한 웹 기반 미니 게임 플랫폼입니다.

### 핵심 특징
- ✅ **21개의 게임**: 다양한 장르의 미니 게임 제공
- ✅ **3개 언어 완전 지원**: 한국어, 영어, 홍콩 번체 중국어
- ✅ **순수 프론트엔드**: HTML, CSS, JavaScript만 사용 (빌드 도구 없음)
- ✅ **반응형 디자인**: 태블릿 및 모바일 최적화
- ✅ **확장 가능한 구조**: 새로운 게임 추가가 용이
- ✅ **Google AdSense 통합**: 광고 수익화 준비 완료

## 🌐 다국어 지원 시스템

### 지원 언어
1. **한국어 (ko)** - 기본 언어
2. **영어 (en)** - English
3. **홍콩 번체 (zh-HK)** - 繁體中文(香港)

### 번역 범위
- ✅ **사이트 전체 UI**: 네비게이션, 버튼, 레이블 등
- ✅ **게임 설명**: 게임 방법, 공략 팁, 게임 소개
- ✅ **게임 내 UI**: 게임별 버튼, 메시지, 안내문
- ✅ **법적 문서**: 개인정보처리방침, 이용약관

### 번역 파일 위치
- `data/i18n/ko.json` - 한국어
- `data/i18n/en.json` - 영어
- `data/i18n/zh-HK.json` - 홍콩 번체

### 언어 선택
- **위치**: 모든 페이지 우측 상단 지구본 아이콘
- **저장**: 로컬 스토리지에 저장되어 다음 방문 시 자동 적용
- **실시간 변경**: 페이지 새로고침 없이 즉시 텍스트 변경

## 📋 게임 개발 규칙

### 필수 사항
1. **window.Game 객체 노출** (IIFE 패턴)
2. **init(gameContainer, options) 메서드 필수**
3. **게임 설명 다국어 지원 필수** (3개 언어 모두)
   - `howToPlay`: 게임 방법
   - `strategy`: 공략 팁
   - `about`: 게임 소개

### 파일 구조
```
games/{game-id}/
├── game.js          # 게임 로직 (필수)
├── game.css         # 게임 스타일 (필수)
└── manifest.json    # 게임 메타데이터 (필수)
```

### manifest.json 예시
```json
{
  "id": "game-id",
  "title": "게임 제목",
  "titleEn": "Game Title",
  "description": "게임 설명",
  "descriptionEn": "Game Description",
  "icon": "🎮",
  "category": "action",
  "difficulty": "easy",
  "version": "1.0.0"
}
```
